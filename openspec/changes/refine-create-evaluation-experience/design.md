## Context

基于现有实现，`/evaluations/new` 已包含：
- 顶部锚点导航（得分球、规则集、规则列表、得分等级）
- 规则集卡片 4 个
- 规则列表表格（规则名称/所属维度只读，分值权重可编辑，支持删除）
- 规则集切换时级联刷新规则列表

当前缺口：
- “立即评估”按钮需要绑定创建评估并跳转详情
- 权重编辑需要明确总和约束
- 删除规则需要最小保留数量约束
- 规则集切换后需要保留未保存改动

## Hard Constraints

1. 页面顶部必须展示“得分球 + 立即评估按钮”。
2. 必须展示 4 个规则集卡片：
   - 数据工程实施指南
   - 数据安全
   - 数据质量
   - 数据治理方法论
3. 规则列表列定义固定：
   - 规则名称（不可编辑）
   - 所属维度（不可编辑）
   - 分值权重（可编辑）
   - 操作（删除）
4. 规则集与规则列表必须级联：默认展示第一个规则集下的规则。
5. 页面必须展示得分等级划分说明。

## Soft Constraints

1. 保持现有左侧导航与右侧主体布局，不引入新路由。
2. 优先复用当前前端数据结构：`API.ruleSets`、`API.rules`、`API.gradeLevels`。
3. 保持页面交互反馈清晰（激活态、空态、编辑态）。

## Dependencies

- 前端数据源：`API.ruleSets`、`API.rules`、`API.gradeLevels`（src/js/api.js）
- 页面逻辑：`renderCreateEvaluation`、`selectRuleSet`、`renderRuleTableRows`、`updateRuleWeight`、`deleteRule`（src/js/app.js）
- 展示样式：规则集卡片、规则表格、锚点导航、得分等级卡片（src/css/style.css）

## Risks

1. 权重编辑无总量约束可能导致“立即评估”输入不合法。
2. 规则删除后若可删空，可能出现“无规则可评估”状态。
3. 规则集切换时若重置用户改动，可能造成误操作感知。

## Confirmed Decisions

1. 点击“立即评估”后：创建评估记录，并跳转评估详情页。
2. 权重约束：当前规则集下分值权重总和必须等于 `100` 才可评估。
3. 删除边界：规则最少保留 `1` 条，不允许删除到 `0`。
4. 规则集切换策略：保留已编辑改动（按规则集分别保留）。

## Validation Rules

1. 分值权重为整数，范围 `0~100`。
2. 点击“立即评估”前校验：当前规则集权重总和必须等于 `100`。
3. 至少保留 1 条规则；删除到 0 条时禁用评估并提示。

## Success Criteria

1. 打开 `/evaluations/new`：首屏可见得分球和“立即评估”按钮。
2. 规则集区域固定展示 4 张卡片，默认首卡激活。
3. 切换规则集后，规则列表内容与所选规则集一致（级联生效）。
4. 规则名称/所属维度不可编辑，分值权重可编辑，删除可用。
5. 评分等级区域展示 A/B/C/D 区间和说明。
6. 点击“立即评估”时，若总权重不为 100，阻断提交并显示提示。
7. 点击“立即评估”时，若校验通过，创建评估记录并跳转到该记录详情页。
8. 删除规则到仅剩 1 条时，删除操作被阻断并有明确提示。
9. 在规则集 A 修改权重后切到 B 再切回 A，A 的修改仍保留。
